3. 우선순위별 개선 과제

우선순위 규칙 현재 상태 해야 할 작업 완료 기준
CRITICAL bundle-dynamic-imports src/App.tsx:7-9에서 주요 라우트 정적 import. 초기 번들 619.37 kB. 라우트 컴포넌트를 React.lazy + Suspense로 분리. 최소 대상: Editor, Reports, Auth. 초기 엔트리 청크가 500 kB 경고 임계치 이하로 감소하거나, 라우트별 분할 청크가 생성되어 경고가 제거될 것.
CRITICAL bundle-conditional 내비게이션 진입 시 편집기 관련 대형 코드가 함께 번들될 가능성 높음 (Editor.tsx 1382 lines). 편집기 전용 UI 블록(예: Export 관련 모달/로직, 대형 패널)을 기능 진입 시점 조건부 로드로 분리. /reports 진입 시 다운로드되는 JS 용량이 명확히 감소하고, 분리 청크가 네트워크 탭에서 확인될 것.
HIGH client-event-listeners resize 리스너가 3개 파일에서 개별 등록됨 (App.tsx:22, Reports.tsx:98, Editor.tsx:427). 공통 훅 1개로 통합하고, 필요한 컴포넌트만 구독. 가능한 경우 CSS media query로 대체해 JS 리스너 자체를 축소. 전역 resize 핸들러 등록 수가 감소하고, 뷰포트 변경 시 불필요 리렌더 횟수가 줄어들 것.
MEDIUM rerender-memo, rerender-no-inline-components Editor.tsx 단일 컴포넌트에 상태/렌더링/네트워크 로직이 집중됨. 좌측 패널/프리뷰/우측 패널을 분리 컴포넌트화하고, props 안정화(useMemo, useCallback) 적용. 입력 1회당 렌더링되는 하위 트리 수가 감소하고, React DevTools 프로파일에서 커밋 시간이 단축될 것.
MEDIUM rendering-content-visibility (적용 후보) 카드 목록 이미지가 즉시 로드됨 (src/pages/Reports.tsx:201). 카드 이미지에 loading="lazy" decoding="async"를 적용하고, 긴 목록 컨테이너에 content-visibility: auto 적용 검토. 초기 스크롤 구간 외 이미지 네트워크 요청이 지연되고, 첫 렌더링 구간의 페인트 비용이 감소할 것.

4. 비적용/판정 보류 항목

  • server-* 카테고리: 본 프로젝트는 Next.js 서버 컴포넌트 기반이 아니므로 직접 적용 대상 아님.
  • async-parallel: 독립 요청 병렬화 여지는 추가 검토 필요. 현재 로그만으로 확정 위반 판정 불가.
  • 실사용 성능 지표(FCP/LCP/INP): 측정 데이터가 없어 확실한 정보 없음.

5. 권장 실행 순서

  1. App.tsx 라우트 lazy 로딩 적용 후 빌드 크기 재측정.
  2. Editor.tsx 분할(패널 단위) + 조건부 로드 적용.
  3. resize 구독 통합 훅 적용 및 기존 개별 리스너 제거.
  4. 리포트 카드 이미지 lazy/decoding 적용.
