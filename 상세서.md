## 1. 제품 범위

### 1.1 MVP 포함 범위

- 차트 이미지 1장 업로드
- OCR + Vision AI 기반 차트 텍스트/수치/추세 추출
- LLM 기반 핵심 요약 3줄 생성
- 웹 에디터에서 템플릿 기반 실시간 프리뷰
- 로고, 카테고리, 출처, 날짜 커스터마이징
- PPTX 파일 다운로드
- 생성 이력 조회 및 재다운로드

### 1.2 MVP 제외 범위

- 모바일 앱 전용 편집 UX
- 차트 2장 이상을 1개 슬라이드에 합성하는 기능
- 팀 협업 편집, 코멘트, 권한 관리
- 사용자가 템플릿 레이아웃을 자유 배치하는 기능
- 영상, GIF, 애니메이션 내보내기

### 1.3 제품 원칙

- 사용자는 5단계 이내에 첫 리포트를 생성해야 한다.
- AI 결과는 자동 반영하되, 신뢰도 낮은 항목은 반드시 사용자 확인을 거친다.
- 기본 출력물은 금융/경제 콘텐츠 제작에 바로 사용할 수 있어야 한다.
- 데스크톱 우선이다. 모바일은 편집 대상이 아니라 안내 대상이다.
- 차트 원본은 왜곡 없이 유지한다. 템플릿 때문에 차트 비율을 강제 변형하지 않는다.

## 3. 핵심 사용자 흐름

### 3.1 신규 리포트 생성

1. 사용자는 `/reports`에서 `새 리포트 만들기`를 클릭한다.
2. 시스템은 임시 `draft` 리포트를 생성하고 `/reports/new`로 이동한다.
3. 사용자는 차트 이미지 1장을 업로드한다.
4. 시스템은 업로드 완료 즉시 OCR/Vision 분석을 시작한다.
5. 분석 완료 후 추출 결과와 AI 요약 3줄을 에디터에 반영한다.
6. 사용자는 카테고리, 로고, 출처, 날짜, 제목 문구를 검토 및 수정한다.
7. 사용자는 우측 프리뷰에서 결과를 확인한 후 `PPTX 내보내기`를 실행한다.
8. 시스템은 export job 완료 후 다운로드 링크를 제공한다.

### 3.2 저신뢰 분석값 보정

1. OCR 또는 추세 추론 confidence가 기준치 미만이면 상태를 `review_required`로 둔다.
2. 해당 필드는 앰버 테두리와 `검토 필요` 배지를 표시한다.
3. 사용자가 값을 수정하거나 `원본 유지`를 선택해야 `완료` 상태로 넘어간다.
4. 검토 전에는 `PPTX 내보내기` 버튼을 비활성화한다.

### 3.3 히스토리 재활용

1. 사용자는 `/reports`에서 이전 리포트를 클릭한다.
2. 시스템은 썸네일, 최종 수정일, 템플릿, export 상태를 보여준다.
3. 사용자는 `재편집` 또는 `다시 다운로드`를 선택한다.
4. export 파일이 만료되었으면 시스템은 동일 데이터로 export job을 다시 실행한다.

### 4.3.1 좌측 입력 패널

#### 섹션 순서

1. 업로드 카드
2. 분석 상태 카드
3. AI 추출값 검토 카드
4. AI 요약 카드

#### 업로드 카드

- 카드 패딩: `20px`
- 드롭존 높이: `220px`
- 허용 파일: `.jpg`, `.jpeg`, `.png`, `.webp`
- 단일 파일 최대 용량: `20MB`
- 최소 해상도: 짧은 변 `960px`
- 권장 해상도: `1600px` 이상
- 다중 파일 업로드: `불가`
- 드롭존 기본 문구: `차트 이미지 1장을 업로드하세요`
- 보조 문구: `권장 1600px 이상, 20MB 이하`
- 썸네일 업로드 후 액션: `교체`, `다시 분석`

#### 분석 상태 카드

| 상태 | 표시 | 버튼 |
| --- | --- | --- |
| `idle` | 업로드 대기 | 없음 |
| `uploading` | 업로드 진행률 바 | 취소 |
| `analyzing` | 단계별 스텝퍼 | 취소 |
| `review_required` | 경고 아이콘 + 검토 필요 수 | `검토 시작` |
| `completed` | 성공 아이콘 + 완료 시간 | `다시 분석` |
| `failed` | 실패 사유 + 재시도 | `재시도` |

#### AI 추출값 검토 카드

- 필드 목록: 차트 제목, 주요 시리즈명, 마지막 값, 기준일, 추세 판정, 단위, 출처 문구
- 각 필드 우측에 confidence badge 표시
- confidence 표시 규칙
  - `0.85 이상`: 녹색 `높음`
  - `0.75 이상 0.85 미만`: 회색 `보통`
  - `0.55 이상 0.75 미만`: 앰버 `검토 필요`
  - `0.55 미만`: 빨간 `수정 필수`
- 입력 필드 높이: `44px`
- 필드 라벨: 12/600
- 필드 설명: 12/400

#### AI 요약 카드

- 요약 개수: 정확히 `3개`
- 각 요약은 기본적으로 `문장 1개`, 최대 `42자`
- 각 요약 줄바꿈 허용: 최대 `2줄`
- 카드 내 액션: `다시 생성`, `전체 복사`, `슬라이드에 반영`
- 사용자가 직접 수정 가능
- 수정 중 길이 초과 시 카운터를 빨간색으로 표시

#### 캔버스 콘텐츠 규칙

- 차트 이미지는 `contain`으로 배치한다.
- 차트 영역 내부 상하 여백은 최소 `12px`를 유지한다.
- 원본 비율 때문에 좌우 빈 공간이 생기면 배경색 `#FFFFFF`로 유지한다.
- 차트 자체에 텍스트가 많은 경우, 확대 렌더링 우선 순위는 차트 영역 > 제목 영역 > 요약 영역이다.
- 헤드라인은 최대 `2줄`, 2줄 초과 시 폰트 크기를 자동 축소한다.
- 자동 축소 순서: `88px -> 80px -> 72px -> 64px`
- 64px에서도 2줄 초과면 말줄임 대신 사용자 수정을 강제한다.

#### 원본 비교 모드

- `원본 비교` 활성화 시 캔버스 하단에 원본 썸네일 strip 노출
- strip 높이: `120px`
- 비교 모드에서도 메인 캔버스 크기는 유지

#### 헤더 설정

| 항목 | 입력 형식 | 제한 |

| 로고 | 이미지 업로드 | PNG/SVG, `2MB` 이하 |
| 브랜드명 | 단일 행 입력 | `16자` 이하 |

- 기본값: AI가 생성한 핵심 헤드라인 1개

## 5. 출력물 규격

### 5.1 PPTX 기본 슬라이드 규격

- PowerPoint 표준 크기: `13.333in x 7.5in`

- 차트 이미지는 원본 비트맵 삽입
- 내보내기 파일명 규칙: `C2I_{category}_{YYYYMMDD}_{slug}.pptx`

### 5.2 PPT 객체 규칙

- 헤드라인, 요약, 출처, 날짜는 모두 텍스트 박스로 삽입한다.

- 폰트 미설치 환경을 고려해 PPT에는 `Pretendard`, `Arial`, `Malgun Gothic` 순서로 fallback 지정

### 5.3 텍스트 오버플로 규칙

- 헤드라인 영역 overflow 발생 시 자동 축소 후에도 넘치면 export 차단
- 요약 bullet overflow 발생 시 줄 수를 2줄로 제한하고 2줄 초과 문장은 사용자 수정 필요
- 출처 overflow 발생 시 말줄임 허용

### 9.1 도메인 엔터티

### Report

### 권장 데이터 구조

```ts
interface Report {

  category: string;

interface AnalysisResult {
  chartTitle: string;

  trendDirection: "up" | "down" | "flat" | "mixed";
  observations: [string, string, string];

  overallConfidence: number;

```

### 9.2 UI 상태

| 상태 키 | 값 | 의미 |
| --- | --- | --- |
| `uploadState` | `idle/uploading/success/error` | 파일 업로드 상태 |
| `analysisState` | `idle/running/review_required/completed/failed` | AI 분석 상태 |
| `editorState` | `clean/dirty/autosaving/save_failed` | 편집 저장 상태 |
| `previewState` | `rendering/ready/error` | 프리뷰 렌더 상태 |
| `exportState` | `idle/queued/processing/ready/failed` | export job 상태 |

### 9.3 상태 전이 규칙

- `draft -> analyzing`: 업로드 성공 직후 자동 전이
- `analyzing -> review_required`: confidence `0.75 미만` 필드 1개 이상 존재
- `analyzing -> ready`: 모든 필드 confidence `0.75 이상`
- `review_required -> ready`: 필수 검토 필드 모두 확인 완료
- `ready -> exporting`: 사용자가 export 실행
- `exporting -> completed`: export job 성공
- `any -> failed`: API, OCR, LLM, render 중 복구 불가 오류 발생

## 11. 접근성 원칙

- 텍스트 대비 비율 `4.5:1` 이상, 큰 텍스트 `3:1` 이상
- 모든 interactive 요소는 키보드 Tab 순서가 시각 순서와 같아야 한다.
- 포커스 스타일은 색상만이 아니라 outline/ring으로도 보여야 한다.
- 업로드 드롭존은 키보드 Enter/Space로도 실행 가능해야 한다.
- 아이콘 단독 버튼은 모두 `aria-label`을 가진다.
- 진행률 변화와 export 완료는 `aria-live="polite"` 영역으로 안내한다.
- 표 기반 히스토리 보기 모드가 있을 경우 썸네일 외 정보를 텍스트로 제공한다.
- 색상만으로 상태를 구분하지 않는다. 아이콘 또는 레이블을 함께 제공한다.

| 차트 다중 감지 | 차트 영역 2개 이상 | `차트가 여러 개 감지되었습니다. 하나만 포함된 이미지를 사용하세요` | 분석 중단 |
| OCR 저신뢰 | confidence `<0.75` | `일부 값의 검토가 필요합니다` | `review_required` 전환 |
| AI 요약 실패 | LLM timeout 또는 응답 스키마 오류 | `요약 생성에 실패했습니다. 다시 생성하세요` | 기존 추출값 유지 |
| 프리뷰 렌더 실패 | 렌더러 예외 | `미리보기를 표시할 수 없습니다` | 원본/텍스트 편집은 유지 |
| export 실패 | 렌더 또는 패키징 실패 | `PPT 생성에 실패했습니다. 다시 시도하세요` | 실패 로그 저장 |
| 세션 만료 | `401/419` | `세션이 만료되었습니다. 다시 로그인하세요` | draft 자동 보존 후 로그인 유도 |
| 저장 실패 | 네트워크 오류 `>10초` | `저장되지 않았습니다. 연결 상태를 확인하세요` | 로컬 draft 캐시 유지 |

## 13. 콘텐츠 규칙

- 카테고리 예시: `미국증시`, `선물시장`, `원자재`, `환율`, `테크`, `매크로`
- 헤드라인은 의문문보다 단정문 우선
- 요약 3줄은 서로 다른 정보여야 한다. 동일 의미 반복 금지
- 요약 문장 시작 금지 패턴: `이 차트는`, `보시다시피`, `전반적으로`
- 출처 문구는 원문 출처가 없으면 빈값 허용
- 날짜 표기 기본 형식: `YY.M.DD`
- 숫자 표기 규칙
  - 천 단위 구분 쉼표 사용
  - 백분율은 소수점 첫째 자리까지
  - 금액/지수/배수는 단위 생략 금지

## 14. 구현 품질 기준

- 첫 프리뷰 렌더 목표: 업로드 후 `8초 이내`
- autosave debounce: `800ms`
- 동일 report의 분석 재실행은 마지막 요청 1건만 유효
- 프리뷰와 export 결과의 텍스트 줄바꿈 차이는 최대 `1줄` 이내
- 프론트엔드에서 optimistic UI를 쓰더라도 server source of truth를 유지한다.

## 15. 문서 적용 우선순위

- 본 문서는 UI/UX 및 출력물 구현의 기준 문서다.
- 개발 중 해석 충돌이 발생하면 본 문서의 수치 규칙을 우선한다.
- 본 문서에 없는 항목은 새로 추가하지 말고 먼저 기준값을 문서에 보완한 뒤 구현한다.
